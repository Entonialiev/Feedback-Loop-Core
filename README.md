# 🔄 Feedback Loop Core (FLC)

**Версия:** 0.1.0  
**Статус:** Спецификация  
**Лицензия:** MIT  

---

## 🧠 Что это?

**Feedback Loop Core (FLC)** — это первая технология, которая заставляет ИИ **оценивать последствия своих действий в реальном времени** и **останавливаться до того, как произойдёт сбой**.

> Современные ИИ-системы работают как «выстрелил и забыл». FLC замыкает контур: действие → оценка → коррекция.

---

## 🔥 Зачем это нужно?

Без FLC агент может:

- Перегрузить API тысячами запросов.
- Внести неверные данные в базу.
- Выполнить опасную команду в системе.
- И **узнать об этом только от человека** (слишком поздно).

С FLC агент:

1. **Выполняет действие.**
2. **Собирает метрики** (время, ошибки, изменения).
3. **Обнаруживает аномалию** (например, ответ за 5 секунд вместо 0.5).
4. **Приостанавливает цепочку** и отправляет сигнал.
5. **Пересматривает решение** с учётом новых данных.

---

## 🏗️ Архитектура

| Слой | Что делает | Как реализовано |
|------|------------|----------------|
| **Сенсорный** | Собирает метрики после каждого вызова | Хуки в API, базах данных |
| **Аналитический** | Находит аномалии через скользящее окно | EWMA, изолирующий лес |
| **Управляющий** | Решает: остановить, скорректировать или игнорировать | Правила на основе порогов |

🛡️ Защита от ложных срабатываний
Адаптивные пороги — норма пересчитывается на основе истории.

Фильтрация — сигналы проверяются через валидатор.

Приоритеты — критичные действия имеют более низкий порог.

📦 Интеграция
LangChain — через кастомный callback.

OpenAI Assistants — через middleware.

Любой HTTP-клиент — через интерцепторы.

Любая ИИ-система — через REST API.

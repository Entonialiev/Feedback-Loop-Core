## 📌 Проблема

Современные ИИ-системы работают по принципу **«запрос → обработка → ответ»**, после чего не отслеживают последствия своих действий. Это приводит к каскадным сбоям, которые система не может предотвратить, так как не получает обратной связи о реальном влиянии на внешнюю среду.

---

## 💡 Решение

**Feedback Loop Core (FLC)** — это встраиваемое ядро, которое:
1. Отслеживает ключевые метрики после каждого действия.
2. Обнаруживает аномалии в реальном времени.
3. Приостанавливает цепочку действий при превышении порогов.
4. Инициирует пересмотр решения с учётом новых данных.

---

## 🏗️ Архитектура

| Слой | Функция | Технология |
|------|---------|------------|
| **Сенсорный** | Сбор метрик (время, код ошибки, дельта состояния) | Хуки в API-клиентах, базах данных |
| **Аналитический** | Обнаружение аномалий через скользящее окно | EWMA, изолирующий лес |
| **Управляющий** | Принятие решения (остановить / скорректировать / игнорировать) | Пороговые правила + пересчёт с новым контекстом |

## 📡 Формат сигнала FLC

```json
{
  "agent_id": "uuid",
  "timestamp": "2026-07-20T12:00:00Z",
  "action_id": "uuid",
  "anomaly_type": "timeout | error_rate | state_delta",
  "severity": 0.85,
  "metrics": {
    "response_time_ms": 1250,
    "error_code": 504,
    "state_change": "+15%"
  }
}


🔄 Процесс работы
Агент выполняет действие.

Сенсорный слой собирает метрики.

Аналитический слой сравнивает с нормой.

При аномалии формируется сигнал.

Управляющий слой приостанавливает цепочку.

Выполняется пересчёт решения.

Цепочка возобновляется с новыми данными (или отменяется).

🛡️ Защита от ложных сигналов
Адаптивные пороги — норма пересчитывается на основе истории.

Фильтрация — сигналы проверяются через валидатор.

Приоритеты — критические действия имеют более низкий порог.

📦 Интеграция
LangChain — через кастомный callback.

OpenAI Assistants — через middleware.

Любой HTTP-клиент — через интерцепторы.
